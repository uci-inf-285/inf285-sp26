A3: Accessibility Browser Extension

In this assignment, you are applying your newfound JavaScript skills to patching some accessibility issues present in webpages.

You will create two extensions targeted at different websites, with a week and a half for each part. In Part 1, you will create an extension to fix a contrived example on a Fake University Website. In Part 2, you will apply these techniques to fix accessibility issues on Wikipedia.

Setting Up and Starter Code

Prior assignments offered flexibility in terms of what browser you used for development. Because we are specifically working on a browser extension, you must use Google Chrome. If you have significant experience with development, you may create your extension for another browser, but please clear it with the course staff first.

The code is available for download on GitHub. Just like with A1 and A2, you can choose to create your own repository or download the code directly.

Repository Structure

The repository contains two folders, fake_university_extension (used in Part 1) and wikipedia_extension (used in Part 2). The two folders are roughly the same, with slight changes to the manifest.json and icons folder to distinguish the two extensions and indicate what website they should apply to. For each part, it's likely that you will only need to edit the make_accessible.js file. There is also a readme.docx to add some context about your submission and reflect on the process. Finally, the helper_functions.js file includes a few JavaScript functions which you might find useful for the assignment, like replacing a tag or wrapping one element with another.

To get started, load the unpacked extension for the particular part from the extensions menu in Chrome. After each time you edit your extension's code, make sure to refresh the extension in this menu as well as your webpage.

Just like for A1 and A2, we recommend Visual Studio Code to edit your files.

Getting Started with Silktide

In addition to downloading the code, you will need to install the Silktide Browser Extension, which is designed to highlight accessibility issues. There are other extensions/tools for identifying issues, but for this assignment we'll rely on and compare against the Silktide report.

Silktide contains a variety of features for testing out accessibility including a screen reader, highlighting focus order, and a color contrast checker. We will focus on the output of the "Accessibility checker" using the Automated test of the WCAG 2.1 AA standard, which is roughly a summary of the others. Note that some of the features may not update as you edit your extension (e.g., the "Alt text" tab is not aware of text added via an extension). But the "Accessibility checker" has been tested, and will follow your extension's edits. You may also find it useful to use other accessibility tools like ANDI, JAWS, or VoiceOver to check accessibility or navigate your page.

As an automated tool, Silktide is often lacking the context necessary to assess whether a potential issue would actually undermine accessibility in practice. Since part of our goal in this assignment is to get practice modifying webpages via code, you will be asked to fix issues surfaced which may or may not be accessibility issues in practice. There will be opportunity to reflect on this distinction in your readme.docx.

DOM Manipulation Advice

This assignment will require digging into JavaScript documentation in order to identify and understand how to use different functions for DOM Manipulation. Much of the assignment can be implemented with functions you used in A2, such as setAttribute and document.getElementBy*. However, you will need to use other features to fix the accessibility issues. A few functions and properties which might prove useful include document.createElement, hasAttribute, appendChild, and style. Beyond DOM methods, ou might also find string includes helpful.
