A1: TODO List in HTML and CSS

In this assignment, you will build a TODO list application in HTML and CSS. Of course, there are many such apps that exist already. But through creating one and customizing its appearance, you'll get some exposure to how webpages are structured and styled.

This assignment is in two parts, with a part due each week. In the first week (Part 1), you will use HTML to edit the default items on the TODO list. In the second week (Part 2), you will use CSS to style your TODO list and add animations.

Starter code

The code is available for download on GitHub.

Repository Structure

We have provided a folder with a number of files to you. You will edit some of them in this assignment, but not all.

The files you should not need to edit are:

  • favicon.ico, which specifies the icon to render in your browser bar. You can choose to change this if you like, such as to make consistent theming in your design. But doing so is not required.
  • script.js. This file includes all of the coding logic for adding and removing items from the TODO list.

Note that this means you should not edit any JavaScript files. You may choose to, but only for adding functionality beyond the assignment requirements.

The files you should edit are:

  • index.html: The HTML file for your TODO list.
  • style.css: The CSS file which indicates your design.
  • readme.docx: A document with some questions to fill out about your experience using a Design System.

Part 1: {{assignments[0].name}}

The TODO list that we've provided offers functionality for adding, completing, and removing events. For the first part of the assignment, you will add some default TODO items. You'll complete your first item, too!

Edit the index.html file to add two default TODO items: one for creating a checklist, a second for styling the webpage, and a third for adding animations. Use your browser's inspector (demoed in lecture) to figure out what tags to use for these items. Once you have added these items, you can add an attribute to the "Create Checklist" item to mark it as complete!

To submit, zip your code repository and upload it to Canvas. Include all files needed to run your webpage.
